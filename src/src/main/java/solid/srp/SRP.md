Single Responsibility Principle (SRP)

A class should have only one responsibility and only one reason to change.

Agar class me multiple responsibilities hain to wo SRP violation hai.

Real life example
Socho ek Restaurant Chef:

Chef ka kaam:
Cook food

Agar chef:
Cook food
Clean table
Manage billing

sab karega → system messy ho jayega.
Isi tarah software me bhi.

Software Example
User system me 3 kaam hote hain:

Save User
Send Email
Generate Report

Agar ye sab ek hi class kare to problem hogi.
